What is JPA?

  • JPA - Java Persistence API, ORM standard specification
  • Maps Java objects to database tables automatically
  • Eliminates boilerplate JDBC code significantly
  • Database-independent (works with any SQL DB)
  • Uses annotations for configuration (@Entity, @Column, etc.)
  • Most popular: Hibernate, EclipseLink, OpenJPA

JPA vs JDBC

  • JDBC: Low-level, manual SQL, verbose code
  • JPA: High-level, object-oriented, minimal boilerplate
  • JPA abstracts database complexity from developers
  • JPA built on top of JDBC drivers internally
  • JPA provides lazy loading, caching, and relationships
  • JDBC needed when JPA can't do specific queries

JPA Architecture

Core JPA Concepts

  • Entity - POJO mapped to database table
  • EntityManager - CRUD operations on entities
  • Persistence Context - Cache of managed entities
  • Transaction - ACID boundary for changes
  • JPQL - SQL-like query language for entities

Basic Entity Class

@Entity @Table(name = "employees") public class Employee { @Id @GeneratedValue(strategy = GenerationType.IDENTITY) private int id; @Column(name = "name", nullable = false) private String name; @Column(name = "salary") private double salary; // Getters and Setters }

Column Annotation Options

  • @Column(name) - Database column name
  • nullable=false - NOT NULL constraint
  • length=100 - VARCHAR length
  • unique=true - UNIQUE constraint
  • columnDefinition - Custom SQL type

EntityManager Operations

  • persist() - INSERT new entity
  • find() - SELECT by primary key
  • merge() - UPDATE existing entity
  • remove() - DELETE entity
  • flush() - Write changes immediately

Persist (INSERT)

EntityTransaction tx = em.getTransaction(); tx.begin(); Employee emp = new Employee(); emp.setName("John Doe"); emp.setSalary(75000); em.persist(emp); // Insert tx.commit(); System.out.println("ID: " + emp.getId());

Find (SELECT)

// Get by primary key Employee emp = em.find(Employee.class, 1); if(emp != null) { System.out.println("Name: " + emp.getName()); System.out.println("Salary: " + emp.getSalary()); } else { System.out.println("Employee not found"); }

Merge (UPDATE)

EntityTransaction tx = em.getTransaction(); tx.begin(); Employee emp = em.find(Employee.class, 1); emp.setSalary(80000); em.merge(emp); // Update tx.commit(); System.out.println("Salary updated to: " + emp.getSalary());

Remove (DELETE)

EntityTransaction tx = em.getTransaction(); tx.begin(); Employee emp = em.find(Employee.class, 1); if(emp != null) { em.remove(emp); // Delete tx.commit(); System.out.println("Employee deleted"); }

JPQL Basics

  • JPQL - SQL-like but operates on entities
  • Uses entity class names, not table names
  • Type-safe (compile time checking)
  • Database-independent query syntax
  • Similar to HQL (Hibernate Query Language)

JPQL SELECT Query

String jpql = "SELECT e FROM Employee e " + "WHERE e.salary > ?1"; Query query = em.createQuery(jpql); query.setParameter(1, 50000); List<Employee> employees = query.getResultList(); for(Employee emp : employees) { System.out.println(emp.getName()); }

Named Queries

@Entity @NamedQuery(name = "Employee.findBySalary", query = "SELECT e FROM Employee e " + "WHERE e.salary > ?1") public class Employee { // ... fields } // Usage Query query = em.createNamedQuery( "Employee.findBySalary"); query.setParameter(1, 50000); List<Employee> list = query.getResultList();

One-to-Many Mapping

@Entity public class Department { @Id private int id; private String name; @OneToMany(mappedBy = "department") private List<Employee> employees; } @Entity public class Employee { @Id private int id; @ManyToOne @JoinColumn(name = "dept_id") private Department department; }

Many-to-One

  • Multiple employees in one department
  • @ManyToOne on owning side
  • @JoinColumn specifies foreign key
  • Most common relationship in databases
  • Eager or lazy loading options

Many-to-Many Mapping

@Entity public class Student { @Id private int id; @ManyToMany @JoinTable(name = "student_course", joinColumns = @JoinColumn(name = "student_id"), inverseJoinColumns = @JoinColumn(name = "course_id")) private List<Course> courses; } @Entity public class Course { @Id private int id; @ManyToMany(mappedBy = "courses") private List<Student> students; }

Cascade Operations

  • PERSIST - Cascade save operation
  • MERGE - Cascade update operation
  • REMOVE - Cascade delete operation
  • ALL - All operations cascade
  • Example: @OneToMany(cascade = CascadeType.ALL)

Loading Strategies

  • LAZY - Load related data when accessed (default)
  • EAGER - Load immediately with parent entity
  • Lazy: better performance, risk of LazyInitException
  • Eager: all data loaded upfront, potential N+1 queries
  • Choose based on usage patterns

Configure Fetch Type

@Entity public class Department { @Id private int id; // Lazy load employees @OneToMany(mappedBy = "department", fetch = FetchType.LAZY) private List<Employee> employees; // Eager load manager @ManyToOne(fetch = FetchType.EAGER) private Manager manager; }

Entity States

  • NEW - Just created, not persisted
  • MANAGED - In persistence context
  • DETACHED - Removed from context
  • REMOVED - Marked for deletion
  • Changes to managed entities auto-sync to DB

JPA Callbacks

@Entity public class Employee { @PrePersist private void prePersist() { System.out.println("Before insert"); } @PostPersist private void postPersist() { System.out.println("After insert"); } @PreUpdate private void preUpdate() { } @PostUpdate private void postUpdate() { } }

Pagination

String jpql = "SELECT e FROM Employee e " + "ORDER BY e.id"; Query query = em.createQuery(jpql); query.setFirstResult(0); // Start row query.setMaxResults(10); // Page size List<Employee> employees = query.getResultList();

JPQL Aggregation

String jpql = "SELECT COUNT(e) FROM Employee e"; Long count = (Long) em.createQuery(jpql) .getSingleResult(); jpql = "SELECT AVG(e.salary) FROM Employee e"; Double avgSalary = (Double) em.createQuery(jpql).getSingleResult(); jpql = "SELECT MAX(e.salary) FROM Employee e"; Double maxSalary = (Double) em.createQuery(jpql).getSingleResult();

Criteria API

  • Type-safe alternative to JPQL
  • Programmatic query building
  • Compile-time checking of attributes
  • More flexible for dynamic queries
  • Slightly more verbose than JPQL

Criteria API Usage

CriteriaBuilder cb = em.getCriteriaBuilder(); CriteriaQuery<Employee> cq = cb.createQuery(Employee.class); Root<Employee> root = cq.from(Employee.class); cq.select(root).where( cb.gt(root.get("salary"), 50000) ); List<Employee> employees = em.createQuery(cq).getResultList();

Transaction Management

  • begin() - Start transaction
  • commit() - Save all changes
  • rollback() - Undo changes
  • Always wrap modifications in transactions
  • Isolation levels affect concurrency

JPA Best Practices

  • Use EntityManager judiciously (stateless)
  • Always handle transactions explicitly
  • Avoid LazyInitException with proper fetching
  • Use named queries for reusable JPQL
  • Implement equals() and hashCode() for entities

Common Pitfalls

  • ❌ Modifying detached entities without merge()
  • ❌ Lazy loading outside transaction scope
  • ❌ N+1 query problem (loading in loops)
  • ❌ Not flushing before queries
  • ❌ Forgetting transaction begin/commit
